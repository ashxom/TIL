
## 익명함수란?

말 그대로 이름 없는 함수.

### 사용하는 이유

나중에 다시 부를 일이 없으므로 프로그램 안에서 일시적으로 한 번만 사용되고 버려져도 되는 객체일 때 사용하는 것.

## 1. 익명 클래스

익명 클래스는 클래스 이름 없이 바로 객체를 생성하는 방법.

인터페이스나 추상 클래스를 구현할 때 주로 쓰임.

ex)

```vhdl
Button btn = new Button();

btn.setOnClickListener(new OnClickListener() {
    @Override
    public void onClick(View v) {
        System.out.println("버튼 클릭");
    }
});
```

### 특징

- 이름이 없는 클래스 → 즉석에서 객체 생성
- 재사용이 불가능함
- 코드가 간결해짐

## 2. 람다식

람다식이란 자바 8부터 등장한 익명 함수 표현식

익명 클래스를 더 간단히 쓰는 방법임.

기본 문법

```vhdl
(매개변수) -> { 실행문 }
```

ex)

```vhdl
// 익명 클래스 방식
Runnable r1 = new Runnable() {
    public void run() {
        System.out.println("실행");
    }
};

// 람다식
Runnable r2 = () -> System.out.println("실행");
```

## 3. 익명 클래스 vs 람다식 비교

| 구분 | 익명 클래스 | 람다식 |
| --- | --- | --- |
| 형태 | 클래스 정의 포함 | 함수형 표현 |
| 코드 길이 | 길다 | 짧고 깔끔함 |
| 등장 버전 | Java 1.1 | Java 8 |
| 사용 대상 | 인터페이스, 추상 클래스 | 함수형 인터페이스 (메서드 1개만) |
| this 의미 | 익명 클래스 자기 자신 | 람다가 속한 외부 객체 |

## 4. 함수형 인터페이스

```vhdl
@FunctionalInterface
interface Greeting {
    void sayHello(String name);
}

Greeting g = (name) -> System.out.println("안녕, " + name);
g.sayHello("DSM");
```

(람다식을 사용하려면 **메서드가 하나만 있는 인터페이스**여야 함.)

## 요약

- 익명 클래스 → **즉석에서 객체를 만들 때**
- 람다식 → **코드를 간결하게 만들 때**
- 함수형 인터페이스 → **람다식의 기반**
