### 인텐트 필터란 무엇인가?

intent filter는 앱의 manifest 파일에 들어 있는 표현으로 앱의 구성 요소가 수신하고자 하는 intent 유형을 나타낸다.

intent 는 명시적 인텐트와 암시적 인텐트로 나뉨

→ 암시적 인텐트를 통해 사용자로 하여금 어떤 앱을 선택할지 할 때 IntentFilter가 필요함.

### 인텐트 필터 생성 방법

본인의 앱 구성요소가 수행할 수 있는 작업에 대하여 별도의 인텐트 필터를 생성하여 다른 앱이나 구성요소가 인텐트를 전달할 수 있도록 해야 함.

인텐트 필터는 앱의 manifest 파일에 있는 <intent-filter> 사이에 정의하며 다음과 같은 세 가지 요소 중 하나 이상을 사용하여 허용할 인텐트 유형을 지정할 수 있음.

- <action>
    - name 특성에서 허용된 인텐트 작업을 선언함.
- <data>
    - 허용된 데이터 유형을 선언함.
- <category>
    - name 특성에서 허용된 인텐트 카테고리를 선언함.
    - 이때 암시적 인텐트를 수신하기 위해서는 CATEGORY_DEFAULT 카테고리를 인텐트 필터에 포함해야 함.

### 요약

1. 인텐트는 다른 구성요소에게 보내는 메시지임
2. 인텐트는 명시적, 암시적으로 나뉘는데 암시적 인텐트에서 인텐트 필터를 사용함
3. 인텐트 필터는 본인이 수행할 수 있는 인텐트를 찾기 위해 명시하는 필터임
